## 1. NeoForge：正统继承者与现代平衡
**适用版本：1.20.1+ (重点 1.21 及更高)**

如果你追求大体量的内容扩展、复杂的机器逻辑或生物群系，**NeoForge** 是目前的工业标准。

* **背景：** 2023 年，由于对原项目领导层的不满，几乎整个 Forge 原开发团队集体“出走”并分叉（Fork）出了 NeoForge。
* **技术特点：**
    * **深度重构：** 彻底清理了 Forge 积累十年的冗余代码，尤其是重写了注册系统和属性附件（Attachment）。
    * **事件系统：** 极其丰富的 Event Bus。在 NeoForge 中，你几乎不需要触碰字节码，通过监听事件就能修改游戏的大部分逻辑。
    * **数据生成（Datagen）：** 强制推行自动化，减少手动编写 JSON 文件的低级错误。
* **现状：** 对于 1.20.4 之后的版本，它已经完全取代了老 Forge 的地位。它的 Gradle 配置更加现代化，编译速度也得到了显著提升。

---

## 2. Fabric：轻量化与速度的极致
**适用版本：全版本（更新极快）**

Fabric 的哲学是“模块化”：它只提供最基础的加载能力，剩下的功能（如渲染、物品组）都交给社区库（Fabric API）。

* **核心优势：**
    * **Mixin 为王：** Fabric 开发者几乎全员“黑客”，通过 Mixin 直接在 Minecraft 源码中插桩。这种方式虽然门槛稍高，但灵活性无可匹敌。
    * **零等待更新：** 当 Mojang 发布预览版（Snapshot）时，Fabric 通常在几小时内就能适配。
    * **Tiny Mappings：** 使用更轻量的混淆映射表，使得内存占用极低，非常适合编写优化类模组（如 Sodium）。
* **开发建议：** 如果你的模组逻辑简单，或者追求极致的启动速度，Fabric 是不二之选。



---

## 3. Quilt：更具野心的社区派系
**适用版本：与 Fabric 兼容**

Quilt 起初是作为 Fabric 的一个分叉诞生的，但它的目标是建立一个更开放、对开发者更友好的生态。

* **特点：**
    * **向下兼容：** 它可以原生运行绝大多数 Fabric 模组。
    * **QSL (Quilt Standard Libraries)：** 相比 Fabric API，Quilt 的标准库封装得更高级，试图解决 Fabric 有时过于“底层”导致开发效率低的问题。
    * **更现代的配置：** 比如原生支持 TOML 或更强大的插件化元数据。
* **现状：** 尽管技术设计非常优雅，但由于和 Fabric 生态重叠较多，目前仍处于“小而美”的状态。

---

## 4. Minecraft Forge：辉煌的落幕
**适用版本：1.7.10 - 1.20.1（经典版）**

曾经的绝对霸主，现在的“遗产维护者”。

* **现状：** 在 1.20.1 之后，随着核心团队转投 NeoForge，原版 Forge 的维护节奏显著放缓。它不再是新技术（如现代渲染优化）的首发平台。
* **为什么还在用：** 为了**存量市场**。如果你要为 1.12.2 或 1.16.5 的大型整合包（如 RL Craft 等）开发续作或补丁，Forge 依然是唯一的战场。它的多线程加载机制在老版本中依然有优势。

---

## 5. Cleanroom：老版本的救星 (1.12.2)
**适用版本：1.12.2 专用**

这是一个非常硬核的项目，旨在通过“换心脏”的方式拯救 1.12.2 这个养老版本。

* **核心功能：**
    * **JVM 升级：** 突破 Java 8 的限制，让 1.12.2 跑在 Java 21 甚至更高版本上，获得显著的 GC 优化。
    * **技术栈翻新：** 引入了 LWJGL3（原版 1.12.2 使用的是已过时的 LWJGL2），这意味着你可以使用更现代的 OpenGL 特性。
    * **Forge 兼容层：** 它是 Forge 1.12.2 的现代化替代品，目标是兼容 99% 的老模组，同时原生支持 Mixin。
* **开发建议：** 如果你专注于 1.12.2 的长线开发，且受够了老 Forge 的内存泄漏和加载缓慢，Cleanroom 是唯一的现代化方案。

---

## 总结：该如何选？

在 2026 年，我的建议如下：
1.  **新项目首选：** **NeoForge**（做大型内容、机器、生态）或 **Fabric**（做功能增强、性能优化）。
2.  **跨平台方案：** 使用 **Architectury SDK**。它允许你编写一套核心代码，同时发布针对 NeoForge 和 Fabric 的版本，极大节省维护成本。
3.  **情怀重燃：** 如果你想拯救那些卡顿的 1.12.2 整合包，请果断尝试 **Cleanroom**。
